# What Kathak Taught Me About Writing Code

People are often surprised when they learn I'm both a software engineer and a Kathak dancer. "Those are so different!" they say. They're wrong.

## Riyaaz and Refactoring

In Kathak, **riyaaz** is daily practice — the repetitive drilling of footwork patterns, spins, and expressions until they become second nature. It's not glamorous. It's showing up every morning and doing the same tukda fifty times until your body remembers it without your brain having to think.

Writing good code is the same. The best engineers I know have a riyaaz — they practice patterns, they refactor obsessively, they build muscle memory for clean architecture. The code that looks effortless is the code that was practiced relentlessly.

## Taal and Type Systems

**Taal** is the rhythmic framework of Kathak. Every movement exists within a cycle — teentaal has 16 beats, jhaptaal has 10, rupak has 7. You can improvise wildly within the taal, but you must always land on **sam** (beat one). Miss the sam, and the whole thing falls apart.

Type systems are taal for code. They're the framework that constrains you just enough to keep you honest. You can be creative, you can be clever, but you must resolve your types. Miss the type, and the whole thing falls apart at runtime.

## Abhinaya and API Design

**Abhinaya** is the expressive storytelling component of Kathak. A great dancer doesn't just execute technically perfect movements — they communicate emotion, narrative, meaning. The audience should *feel* something.

Good API design is abhinaya. It's not enough for an API to work — it should communicate intent. The naming should tell a story. The structure should feel intuitive. The developer using your API should understand what it does without reading the docs (but you should still write the docs).

## The Real Intersection

The deepest parallel is this: both Kathak and software engineering reward the combination of **discipline and creativity**. Pure discipline without creativity gives you correct but lifeless code, or technically perfect but emotionally empty dance. Pure creativity without discipline gives you unmaintainable code, or beautiful ideas that never quite land on beat.

The magic is in the intersection. And that's where I try to live.
